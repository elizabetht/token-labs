# Flux GitOps Setup

This directory contains the Flux CD configuration for managing the Kubernetes cluster in a GitOps manner.

## Overview

Flux automates the deployment and management of Kubernetes operators and applications by continuously reconciling the cluster state with this Git repository.

## Directory Structure

```
deploy/
├── flux-system/              # Flux system components (generated by bootstrap)
│   ├── namespace.yaml
│   ├── gotk-sync.yaml        # GitRepository source
│   └── gotk-kustomization.yaml
├── infrastructure/
│   ├── sources/              # HelmRepository and GitRepository sources
│   │   ├── helm-repositories.yaml
│   │   └── git-repositories.yaml
│   └── controllers/          # HelmRelease manifests for each operator
│       ├── gateway-api-crds.yaml
│       ├── longhorn.yaml
│       ├── gpu-operator.yaml
│       ├── envoy-gateway.yaml
│       ├── kuadrant.yaml
│       └── kube-prometheus-stack.yaml
├── cluster/                  # Cluster-specific manifests (NVIDIA, etc.)
├── gateway/                  # Gateway API resources
├── llm-d/                    # llm-d deployment
├── policies/                 # Kuadrant policies
└── tenants/                  # Tenant configurations
```

## Prerequisites

1. Kubernetes cluster bootstrapped via Ansible (see `../ansible/`)
2. GitHub personal access token with repo permissions
3. Flux CLI installed on your local machine

### Install Flux CLI

```bash
curl -s https://fluxcd.io/install.sh | sudo bash
```

## Bootstrap Flux

### 1. Set up GitHub token

```bash
export GITHUB_TOKEN=<your-github-token>
export GITHUB_USER=elizabetht
export GITHUB_REPO=token-labs
```

### 2. Bootstrap Flux into the cluster

```bash
flux bootstrap github \
  --owner=${GITHUB_USER} \
  --repository=${GITHUB_REPO} \
  --branch=main \
  --path=./deploy \
  --personal
```

This command will:
- Install Flux components in the `flux-system` namespace
- Create a GitRepository resource pointing to this repo
- Create a Kustomization resource to sync `./deploy` path
- Configure Flux to automatically apply changes from Git

### 3. Verify Flux installation

```bash
flux check
kubectl get pods -n flux-system
```

## Deployed Components

Flux will automatically deploy and manage:

### Infrastructure Layer

1. **Gateway API CRDs** (v1.4.1)
   - Standard Gateway API resources
   - Inference Extension CRDs (v1.3.0)
   - Envoy AI Gateway CRDs (v0.5.0)

2. **Longhorn** (distributed storage)
   - Default StorageClass
   - 2 replicas per volume
   - Web UI for management

3. **NVIDIA GPU Operator**
   - GPU device plugin
   - DCGM exporter for GPU metrics
   - MIG manager
   - Node status exporter

4. **Envoy Gateway** (v1.6.x)
   - L7 proxy and ingress
   - Extension API support

5. **Kuadrant Operator**
   - Authorino (authentication)
   - Limitador (rate limiting)
   - Redis for rate limit storage

6. **kube-prometheus-stack**
   - Prometheus for metrics
   - Grafana for dashboards
   - Alertmanager for alerts
   - Pre-configured to scrape:
     - DCGM GPU metrics
     - Limitador token usage metrics

### Application Layer

Additional components in other directories:
- `cluster/` - NVIDIA DRA driver, GPU sharing configs
- `gateway/` - Gateway and AIGatewayRoute resources
- `llm-d/` - Inference scheduler and model services
- `policies/` - AuthPolicy, RateLimitPolicy, TokenRateLimitPolicy
- `tenants/` - Tenant API keys and quotas

## Managing Deployments

### Check reconciliation status

```bash
flux get all
flux get helmreleases -A
flux get kustomizations -A
```

### Force reconciliation

```bash
flux reconcile source git flux-system
flux reconcile kustomization flux-system
```

### Suspend/Resume a HelmRelease

```bash
# Suspend
flux suspend helmrelease longhorn -n flux-system

# Resume
flux resume helmrelease longhorn -n flux-system
```

### View logs

```bash
# Flux controller logs
flux logs --level=info --all-namespaces

# Specific HelmRelease reconciliation
kubectl logs -n flux-system deploy/helm-controller -f
```

## Secrets Management with SOPS

For production, encrypt secrets using SOPS with age encryption:

### 1. Generate age key

```bash
age-keygen -o age.agekey
```

### 2. Create Kubernetes secret with age key

```bash
cat age.agekey |
kubectl create secret generic sops-age \
  --namespace=flux-system \
  --from-file=age.agekey=/dev/stdin
```

### 3. Configure Flux to use SOPS

Add to `gotk-kustomization.yaml`:

```yaml
spec:
  decryption:
    provider: sops
    secretRef:
      name: sops-age
```

### 4. Encrypt secrets

```bash
# Create .sops.yaml in repo root
cat <<EOF > .sops.yaml
creation_rules:
  - path_regex: .*.yaml
    encrypted_regex: ^(data|stringData)$
    age: <your-age-public-key>
EOF

# Encrypt a secret
sops --encrypt --in-place secret.yaml
```

## Operator Dependencies

The operators are deployed in order based on dependencies:

1. **Gateway API CRDs** - Must be installed first
2. **Longhorn** - Provides storage for other components
3. **GPU Operator** - Independent, can deploy in parallel with Longhorn
4. **Envoy Gateway** - Depends on Gateway API CRDs
5. **Kuadrant** - Depends on Gateway API CRDs
6. **kube-prometheus-stack** - Depends on Longhorn (for storage), GPU Operator (for DCGM metrics)

Dependencies are managed via `dependsOn` fields in HelmRelease manifests.

## Customization

### Override HelmRelease values

Edit the `values:` section in the HelmRelease files:

```yaml
# deploy/infrastructure/controllers/longhorn.yaml
spec:
  values:
    defaultSettings:
      defaultReplicaCount: 3  # Change from 2 to 3
```

Commit and push changes. Flux will automatically reconcile.

### Add a new operator

1. Add HelmRepository to `deploy/infrastructure/sources/helm-repositories.yaml`
2. Create HelmRelease in `deploy/infrastructure/controllers/<name>.yaml`
3. Add to `deploy/infrastructure/controllers/kustomization.yaml`
4. Commit and push

## Troubleshooting

### HelmRelease fails to install

```bash
# Check HelmRelease status
kubectl describe helmrelease <name> -n flux-system

# Check helm-controller logs
kubectl logs -n flux-system deploy/helm-controller --tail=100
```

### Source fails to fetch

```bash
# Check GitRepository or HelmRepository
flux get sources all

# Check source-controller logs
kubectl logs -n flux-system deploy/source-controller --tail=100
```

### Kustomization fails to apply

```bash
# Check Kustomization status
kubectl describe kustomization flux-system -n flux-system

# Check kustomize-controller logs
kubectl logs -n flux-system deploy/kustomize-controller --tail=100
```

### Force reconciliation after fixing issues

```bash
flux reconcile helmrelease <name> -n flux-system --with-source
```

## Uninstall

To remove Flux and all managed resources:

```bash
# Suspend all kustomizations to prevent reconciliation
flux suspend kustomization flux-system

# Uninstall Flux
flux uninstall
```

**Warning**: This will remove all Flux components but NOT the deployed operators. To fully clean up, manually delete the operator namespaces and CRDs.
